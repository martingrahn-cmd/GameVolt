# GameVolt.io — SEO Cleanup: Ghost Pages & www Redirect

## Background

GameVolt.io was previously a different site with a `/game/` URL structure. When the site was rebuilt as an original HTML5 games platform (hosted on GitHub Pages), the old `/game/` routes were removed — but Google still has them indexed. These ghost pages are hurting SEO by making the site look like it has broken/thin content.

Additionally, Google treats `www.gamevolt.io` and `gamevolt.io` as two separate sites, splitting authority between them.

---

## Task 1: Block and remove ghost `/game/` pages

Google currently indexes these URLs that should NOT exist:

- `/game/squidgame-multiplayer` (13 impressions)
- `/game/dogecoin-clicker` (11 impressions)
- `/game/fireboy-and-watergirl-online` (11 impressions)
- `/game/ninja-veggie-slice` (2 impressions)
- `/game/resident-evil-purge-operation` (1 impression)

There may be more. These are from the old site and have nothing to do with the current GameVolt.

### Option A — If `/game/` routes are generated by code/config:
Find and remove whatever generates these routes. Check for:
- A data file (JSON/YAML) listing old games
- A template that auto-generates `/game/[slug]` pages
- Any CMS or static site generator config that creates these paths

### Option B — If the pages don't actually exist (404s that Google remembers):
Create a `/game/index.html` catch-all that returns proper signals to Google:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta name="robots" content="noindex, nofollow">
  <link rel="canonical" href="https://gamevolt.io/" />
  <meta http-equiv="refresh" content="0;url=https://gamevolt.io/">
  <title>Redirecting...</title>
</head>
<body>
  <p>This page has moved. <a href="https://gamevolt.io/">Go to GameVolt.io</a></p>
</body>
</html>
```

This tells Google: don't index this, the canonical is the homepage, and redirect users there.

### Also add to robots.txt:
```
Disallow: /game/
```

### After deploying — manual step (NOT for Claude Code):
Go to Google Search Console → Removals → New Request, and submit these URLs for temporary removal:
- `gamevolt.io/game/squidgame-multiplayer`
- `gamevolt.io/game/dogecoin-clicker`
- `gamevolt.io/game/fireboy-and-watergirl-online`
- `gamevolt.io/game/ninja-veggie-slice`
- `gamevolt.io/game/resident-evil-purge-operation`

This speeds up de-indexing from weeks to days.

## Task 2: Fix www vs non-www split

Google sees `www.gamevolt.io` and `gamevolt.io` as separate sites. Currently:
- `gamevolt.io` gets 316 impressions, 17 clicks
- `www.gamevolt.io` gets 196 impressions, 4 clicks

We want everything on `gamevolt.io` (without www).

### For GitHub Pages:
GitHub Pages doesn't support server-side redirects, so we handle this via DNS and canonical tags.

#### Step 1: Ensure canonical tags point to non-www
Check that EVERY page on the site has this in `<head>`:
```html
<link rel="canonical" href="https://gamevolt.io/[current-path]" />
```

Make sure no page has `www.gamevolt.io` in its canonical tag. Search the entire codebase for `www.gamevolt.io` and replace with `gamevolt.io`.

#### Step 2: Check CNAME file
The `CNAME` file in the repo root should contain:
```
gamevolt.io
```
NOT `www.gamevolt.io`.

#### Step 3: DNS configuration (manual step — NOT for Claude Code)
In the domain registrar (probably where gamevolt.io was bought), ensure:
- `A` records point to GitHub Pages IPs (185.199.108-111.153)
- `CNAME` record for `www` points to `gamevolt.io` or the GitHub Pages URL
- GitHub should auto-redirect www → non-www if configured correctly

Check in GitHub repo → Settings → Pages that:
- Custom domain is set to `gamevolt.io`
- "Enforce HTTPS" is checked
